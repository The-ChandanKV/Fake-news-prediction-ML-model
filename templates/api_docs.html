FakeDetect AI — API Reference

A production-grade REST API for real-time fake news detection, content authenticity analysis, IPR originality scoring, and multi-language fact verification. Integrate it into any media platform, browser extension, or content moderation system.

BASE URL https://fakedetect-ai.up.railway.app
⚡

High Performance

Predictions return in under 300ms with full explainability data included

🌍

45+ Languages

Auto-detect and translate non-English articles before classification

🔓

Open Integration

Standard JSON REST API. No SDK required. Works from any stack.

🔑 Authentication

The current version is open-access for demonstration purposes. In production, pass your API key in the request header:

// Request Header X-API-Key: your_api_key_here Content-Type: application/json
⚠️ Error Handling

All API errors return a JSON body with an error field and a standard HTTP status code.

CodeMeaning
200Success
400Bad Request — missing or invalid parameters
503Model not loaded — run train_improved_model.py
500Internal Server Error
📡 Endpoints
POST Full Prediction Analysis /api/predict Core endpoint ↓

Analyzes a news article and returns a full prediction with confidence score, explainability heatmap, fact-check results, source credibility, language detection, and trust score.

Request Body
ParameterTypeDescription
textstringThe news article text to analyze (min 10 chars)
Example Request
// JavaScript fetch const response = await fetch('/api/predict', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ text: "Scientists have discovered a cure for all cancers using common household bleach..." }) }); const data = await response.json();
Response 200 OK
{ "prediction": "Fake News", "confidence": 94.7, "label": 1, "overall_trust_score": 12.3, "language": { "detected": "en", "name": "English", "translated": false }, "sources": [], "fact_checks": [ { "claim": "...", "rating": "False", "publisher": "Snopes", "url": "..." } ], "explanation": { "fake_indicators": [ { "word": "bleach", "impact": 0.84 }, ... ], "real_indicators": [ ... ] }, "analyzed_at": "2026-04-16T09:30:00" }
POST Word Importance Heatmap /api/heatmap LIME-style ↓

Returns the article text annotated token-by-token with a score (−1.0 to +1.0). Positive scores indicate fake-leaning words (shown in red), negative scores indicate real-leaning words (shown in green). Use this to build an inline text heatmap UI.

POST /api/heatmap { "text": "Article text here..." } // Response { "success": true, "annotated": [ { "token": "Scientists", "score": -0.12 }, { "token": "bleach", "score": +0.67 }, ... ] }
POST IPR Content Originality Check /api/ipr-check Unique to FakeDetect ↓

Analyzes the content for Intellectual Property Rights (IPR) concerns — measures vocabulary diversity, detects copy-paste writing patterns, identifies sensationalized language, and returns an originality score (0–100).

ParameterTypeDescription
textstringArticle text (min 30 chars)
check_onlinebooleanAlso check key phrases against web (slower, default: false)
// Response { "ipr": { "originality_score": 62, "ipr_risk": "Medium Risk — Some phrases may be sourced", "ipr_color": "yellow", "vocab_diversity": 58.3, "key_phrases": [ "scientists discover cure", ... ], "ipr_summary": { "total_words": 320, "unique_words": 187, "pattern_flags": [ "Excessive capitalization detected" ] } } }
GET Real-Time Trending Topics /api/trending Live dashboard ↓

Fetches current trending headlines from BBC and NYT via RSS, runs each through the fake news model, and returns them together with a fake probability and risk level. No parameters required.

GET /api/trending // Response { "success": true, "topics": [ { "title": "Global leaders meet at UN summit...", "category": "World", "source": "BBC", "prediction": "Real", "fake_probability": 12.4, "risk_level": "Low", "link": "https://..." } ] }
GET Platform Analytics Stats /api/stats Dashboard data ↓

Returns aggregated statistics: total analyses, fake vs real split, 7-day trend, top flagged domains, language distribution, and feedback accuracy. Powers the Analytics Dashboard.

GET /api/stats // Returns full dashboard stat object (see dashboard.html)
POST Submit User Feedback /api/feedback Continuous learning ↓

Submit whether a prediction was correct or incorrect. This feedback is stored and incorporated into the next model retraining cycle, enabling continuous improvement.

ParameterTypeDescription
textstringThe original article text
label0 or 1Predicted label (0=Real, 1=Fake)
correctbooleanWas the prediction correct?
POST Generate Share Card /api/share-card WhatsApp / Social ↓

Generates a formatted verification report as a shareable text string optimized for WhatsApp, Twitter, and other social platforms. Returns a pre-encoded WhatsApp deep-link URL.

POST /api/share-card { "prediction": "Fake News", "confidence": 94.7, "overall_trust_score": 12, "language": { "name": "English" }, "text_preview": "Scientists claim bleach cures cancer..." } // Response { "share_text": "🚨 *FakeDetect AI Verdict*\n━━━━━━━━━━━━━━\n...", "whatsapp_url": "https://wa.me/?text=..." }
GET Health Check /health Status ↓

Returns the platform health status, model load state, version number, and list of enabled features. Use this to verify the API is operational before making prediction requests.

GET /health // Response { "status": "healthy", "model_loaded": true, "version": "3.0.0", "features": [ "multi_language", "fact_check", "ipr_check", "heatmap", "trending" ] }
